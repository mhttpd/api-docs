Class MiniFCGI_Manager

Description
Description | Vars (details) | Methods (details) Constants (details)

The MiniFCGI manager class.

This static class is responsible for managing the FastCGI process pool. It connects clients to available processes, spawns new ones if needed, monitors activity on each of the processes and maintains a scoreboard for the pool.

  • author: MiniHTTPD Team
  • copyright: (c) 2010 MiniHTTPD Team
  • todo: Add better dynamic management of the process pool, particularly for culling idle or zombie processes.

		static connect (line 162)
	
 
	
	

Connects a client to a running process, or spawns a new one if needed.


The method will try to handle new client connections in the most optimal  way, otherwise if a client is trying to resume a connection to an already  existing process (for example, when reading an FCGI response) and cannot  complete the connection, the method must abort, as responses need to come  from the same process as the original requests.

	
    
				
  • return: the socket connection or error

		static spawn (line 110)
	
 
	
	

Adds new FastCGI processes to the pool.


At startup, this method will spawn the minimum number of processes given  in the configurations settings. After startup, ID numbers should be passed by  clients when requesting a new process. Using this method ensures that the  maximum number of processes given in the configuration is never exceeded.
